# Linked Wiki Pages – Azure DevOps Extension

An Azure DevOps extension that displays linked wiki pages on work items and automatically scans wikis to create those links.

## Features

- **Linked Wiki Pages panel** — Shows all wiki pages linked to the current work item directly on the work-item form
- **Update Wiki Links button** — One-click button on each work item to scan all wikis for references to that work item and auto-create missing links
- **Wiki Auto-Link Scanner hub** — Bulk scanner under Boards that scans every wiki page for `#NNN` references, shows discovered links with checkboxes, and lets you choose which to create

## Getting Started

```bash
cd linked-wiki-pages
npm install
tfx extension create --manifest-globs vss-extension.json
```

Then upload the `.vsix` to your Azure DevOps organization via **Organization Settings → Extensions**.

## How It Works

The scanner reads wiki pages from the backing Git repository via the VSS SDK REST clients (`GitRestClient`, `WorkItemTrackingRestClient`). It detects work-item references in two forms:
- `#123` — standard markdown references
- `/_workitems/edit/123` — full URL references (auto-generated by the wiki editor)

For each reference found, it creates an `ArtifactLink` of type `Wiki Page` on the work item.
